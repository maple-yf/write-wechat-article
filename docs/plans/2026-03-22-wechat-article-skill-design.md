# 微信公众号文章写作技能设计文档

**项目名称**：write-wechat-article
**设计日期**：2026-03-22
**版本**：v1.0

---

## 一、项目概述

### 1.1 目标

为 AI Agent（Claude Code、OpenCode、OpenClaw）开发一个微信公众号文章写作技能，帮助个人博主快速生成高质量、符合微信公众号格式的文章。

### 1.2 核心功能

- AI 自动生成文章
- 文章润色优化
- 微信公众号排版美化
- 吸引人的标题生成
- 封面图和文章插图生成

### 1.3 目标用户

个人博主，需要快速产生高质量内容。

---

## 二、架构设计

### 2.1 架构类型

**模块化多技能架构** - 主技能协调 + 子技能执行

### 2.2 技能结构

```
write-wechat-article/
├── skills/
│   ├── write-wechat-article/      # 主技能（协调入口）
│   ├── wechat-generate/           # 生成子技能
│   ├── wechat-polish/             # 润色子技能
│   ├── wechat-format/             # 排版子技能
│   └── wechat-title/              # 标题子技能
├── shared/                         # 共享模块
│   ├── prompts.js                 # Prompt 模板
│   └── format-utils.js            # 格式工具函数
├── scripts/                        # 执行脚本
│   ├── install.js
│   ├── build.js
│   └── test.js
└── output/                         # 默认输出目录（可配置）
```

### 2.3 工作模式

1. **创作流程**：串联所有子技能，生成完整文章
2. **修改流程**：按意图调用对应子技能，修改指定内容

---

## 三、组件设计

### 3.1 主技能（write-wechat-article）

**职责**：意图解析和任务分发

**输入**：用户自然语言指令
**输出**：调用相应子技能并返回结果

### 3.2 子技能组件

| 子技能 | 输入 | 输出 |
|-------|------|------|
| wechat-generate | 主题、字数、风格 | Markdown 内容（含插图位置标记） |
| wechat-polish | 待优化文章、优化方向 | 优化后的文章 |
| wechat-format | Markdown 文章 | 微信公众号 HTML、封面图 HTML |
| wechat-title | 文章内容或摘要 | 候选标题列表 |

### 3.3 共享模块

**prompts.js**：
- 预定义的 prompt 模板
- 文章结构模板
- 封面图设计规范

**format-utils.js**：
- Markdown 转微信公众号 HTML 工具函数
- 封面图 HTML 生成函数
- 格式验证函数

---

## 四、数据流设计

### 4.1 创作流程

```
用户输入 → 意图解析 → 参数提取 → wechat-generate → wechat-title
→ wechat-polish → wechat-format → 输出文件
```

### 4.2 修改流程

```
用户输入 → 意图解析 → 参数提取 → 单步调用子技能 → 输出更新文件
```

### 4.3 插图嵌入机制

**插图数量控制**：
- 短文章（<1500 字）：1-2 张
- 中等文章（1500-3000 字）：3-4 张
- 长文章（>3000 字）：4-5 张（上限 5 张）

**插图位置**：章节开头/结尾、重要观点之后、对比内容处

**插图类型**：对比图、核心能力图、流程图

## 五、输出产物

### 5.1 文件类型

| 文件 | 说明 |
|-----|------|
| `.md` | 文章源文件（Markdown） |
| `-wechat.html` | 可直接粘贴到微信公众号编辑器的 HTML |
| `cover-.html` | 封面图 HTML（浏览器打开截图使用） |

### 5.2 输出位置

**默认**：`./output/[文章slug]/`

**可配置**：
- 调用参数指定
- 环境变量
- 配置文件

## 六、微信公众号兼容性规范

### 6.1 HTML 标签规范

- ✅ 使用 `<section>` 标签
- ✅ 使用 `<p>` 标签代替 `<ul>`/`<li>`
- ✅ 表格标题背景色应用到每个 `<th>` 标签
- ❌ 禁止 `<style>` 标签
- ❌ 禁止 `<div>` 标签

### 6.2 主题色

**主题色**：`rgb(0, 112, 192)` 或 `#0070c0`

**标题样式**：
- H1：居中 + 主题蓝背景 + 白色字体
- H2：主题蓝色 + 下边框
- H3：深灰色 + 加粗

### 6.3 封面图规范

- 左侧矩形：1175×500px
- 右侧方形：500×500px
- 设计风格：科技蓝主题 + 玻璃拟态效果

---

## 七、错误处理

### 7.1 错误类型

| 类型 | 处理方式 |
|-----|---------|
| 输入验证错误 | 提示用户提供缺失参数 |
| 文件操作错误 | 提示权限或磁盘空间问题 |
| 格式转换错误 | 自动过滤不兼容属性，记录警告 |
| Agent 调用错误 | 提供重试选项 |

### 7.2 日志记录

**位置**：`./output/[文章slug]/.skill.log`

**内容**：操作时间戳、输入参数、执行步骤、错误信息、输出文件路径
